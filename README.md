====== 관련 정리 (previwe) =======  

React : 사용자 인터페이스를 만드는 프론트엔드 라이브러리로, 서버와 통신해 데이터를 화면에 표시하는 역할을 함  
REST API : 백엔드에서 HTTP 기반으로 데이터를 주고받기 위해 사용하는 표준 인터페이스 방식   
JPA (Java Persistence API) : 자바 객체(Entity)를 데이터베이스 테이블에 매핑해 SQL 없이 DB 작업을 가능하게 하는 ORM 기술  
Service 레이어 : 비즈니스 로직을 처리하는 계층으로, 컨트롤러와 DAO 사이에서 핵심 처리를 담당  
DTO (Data Transfer Object) : 클라이언트와 서버 사이에서 데이터를 주고받기 위한 전용 객체로, Entity와 분리하여 사용  
Map (매핑) : DTO와 Entity 사이의 데이터를 변환하는 작업으로, 수동으로 하거나 MapStruct 같은 라이브러리를 사용할 수 있음  
ResponseEntity : HTTP 상태 코드, 헤더, 본문을 직접 설정해 응답을 보다 세밀하게 제어할 수 있게 해주는 클래스  
DAO (Data Access Object) : 데이터베이스와의 직접적인 입출력 작업을 수행하는 계층, 보통 JPA에서는 @Repository로 정의  
TDD (Test Driven Development) : 기능 구현 전에 테스트 코드를 먼저 작성하고, 그에 맞춰 코드를 개발하는 방식  
Exception (예외 처리) : 오류 발생 시 사용자에게 적절한 메시지와 상태 코드를 반환하도록 처리하는 방법, 보통 @ControllerAdvice와 함께 사용  
Setter : 객체 생성 후 필드를 메서드를 통해 설정하는 방식이지만, 불변성이 약해질 수 있음  
Builder 패턴 : 메서드 체이닝 방식으로 가독성 있게 객체를 생성하는 패턴으로, Lombok의 @Builder로 자주 사용됨   
Validator : 사용자 입력값의 유효성을 검사하는 기능으로, @NotBlank, @Size 등의 어노테이션을 활용함  
Kotlin : 자바보다 간결하고 안전한 문법을 가진 현대적인 JVM 언어로, Spring Boot와 완전 호환됨  
WebFlux : 스프링의 비동기(Non-blocking) 웹 프레임워크로, 많은 요청을 효율적으로 처리할 수 있어 실시간 서비스에 적합함
