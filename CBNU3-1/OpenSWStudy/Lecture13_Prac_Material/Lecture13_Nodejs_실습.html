Introduction to Node.js 개념 · 구현 없음

개념 정리 [1~10p]

  • 백엔드 = 서버 측. 데이터·로직·인증·DB 통신 담당 [4p]
  • Node.js = V8 엔진 + 서버 기능 → 브라우저 밖에서 JS 실행 [5p]
  • Node로 할 수 있는 것: 서버 생성 / DB 연결 / 파일 입출력 / 인증 / REST API / React에 데이터 전송 [6p]
  • 3대 핵심 아이디어 [7p]
    • 비동기(Asynchronous): 한 작업이 끝나길 기다리지 않음
    • 논블로킹 I/O(Non-Blocking): 대기 중에도 다음 요청 처리
    • 이벤트 드리븐(Event-Driven): 이벤트 발생 시 콜백 실행
  • 내부 동작: V8 → Node API → libuv(이벤트 루프 + 워커 스레드) [8p]
  • 적합: I/O 중심(REST API, 로그인, 채팅, 실시간) [9p]
  • 부적합: CPU 중심(ML 학습, 영상 인코딩, 대규모 연산) → 단일 스레드 이벤트 루프가 막힘 [10p]

PART 2

Node.js 생태계 & 코어 모듈

PRACTICE 1 프로젝트 셋업 & 코어 모듈 [11~22p]

  • npm = 패키지 매니저, package.json = 프로젝트 설정(의존성·scripts), node_modules = 설치된 패키지 폴더 [12~16p]
  • 코어 모듈(설치 불필요, require만): fs(파일), path(경로), http(서버) [17p]
  • fs.writeFile(파일, 내용, 콜백) [18p] / path.join(__dirname, ...) [20p] / http.createServer(...).listen(3000) [22p]

🛠️ TASK 0 프로젝트 초기화

할 일
  1. 작업 폴더에서 npm init -y 실행 → package.json 생성 [15p]
  2. npm install express, npm install mysql 설치 [13p, 46p]
  3. package.json의 dependencies에 express, mysql이 잡혔는지 확인 [16p]
  4. package.json의 "scripts"에 "start": "node server.js" 추가 [15~16p]
예상 결과
package.json 의 dependencies:
  "express": "^5.x.x",
  "mysql":   "^2.x.x"
node_modules/ 폴더와 package-lock.json 생성됨

🛠️ TASK 1 코어 모듈로 "파일 로거 + 기본 서버"

할 일 (파일: task1_core.js)
  1. fs로 log.txt에 "Server started" 기록 [18p]
  2. 그 경로를 path.join(__dirname, "log.txt")로 만들기 (하드코딩 금지) [20p]

// 3) 서버 생성

  /* TODO: res.end(...) */

server.listen(/* TODO */);

PART 3

Express.js

PRACTICE 2 Express 기본 & 라우팅 [23~28p]

  • Express = Node 위의 웹 프레임워크. 메서드 검사·URL 매칭·응답 포맷을 자동 처리 [24~25p]
  • 최소 구조: require("express") → const app = express() → app.get(...) → app.listen(3000) [26p]
  • req(요청 정보) / res(응답: res.send(...)) [26p]
  • 라우팅 = URL을 핸들러 함수에 매칭. 정의 순서대로 매칭 [27~28p]

🛠️ TASK 2 기본 서버를 Express로 다시 만들기

할 일 (파일: server.js — 이후 과제는 모두 이 파일에 누적)
  1. Task1의 raw http 서버를 Express로 교체 [25~26p]
  2. 라우트 3개 구현 [28p]
    • GET / → "Home Page"
    • GET /products → "Product List"
    • GET /users → "User List"
  3. 서버 시작 시 콘솔에 "Server is running on port 3000" 출력 [26p]
예상 결과
http://localhost:3000/          → Home Page
http://localhost:3000/products  → Product List
http://localhost:3000/users     → User List
http://localhost:3000/xxx       → (Express 기본) Cannot GET /xxx   ← 이제 URL을 구분함!
셀프 체크: raw http 버전(Task1) 대비 코드가 짧아진 이유는? [25p]

PRACTICE 3 HTTP 메서드 / URL 파라미터 / 쿼리스트링 [29~37p]

  • GET(조회) / POST(생성) / PUT(수정) / DELETE(삭제) [29~32p]
  • URL 파라미터 :id → req.params.id (필수값, 동적 라우트) [33~35p]
  • 쿼리스트링 ?key=value → req.query.key (선택값: 검색·필터·정렬) [36~37p]

🛠️ TASK 3 User/Product API 라우트 확장

할 일 (server.js에 추가)
  1. GET /users/:id → "User ID is " + id (params) [34p]
  2. GET /products?category=phone → "Category is phone" (query) [36p]
  3. POST /login → "Login data received" (본문 처리는 Task4에서) [30p]

PRACTICE 4 미들웨어 & express.json() [38~41p]

  • 미들웨어 = 요청과 응답 사이에서 실행되는 함수. next()로 다음 단계로 넘김 [38~39p]
  • app.use(fn) = 모든 라우트에 적용(로깅·인증·파싱) [39p]
  • React는 데이터를 JSON 텍스트로 보냄 → app.use(express.json())가 파싱해서 req.body 객체로 만듦. 없으면 req.body는 undefined [40~41p]

PART 4

Node.js & MySQL

PRACTICE 5 DB 개념 & 연결 [42~52p]

  • DBMS / 관계형 모델: DB = 테이블(릴레이션)들의 집합, 행=튜플, 열=속성, 기본키=중복 없는 식별자 [43~45p]
  • 드라이버 설치: npm install mysql → var mysql = require("mysql") [46p]
  • 연결: mysql.createConnection({host, user, password, database}) → con.connect(cb) [47, 51p]
  • 쿼리 실행: con.query(sql, function(err, result){ ... }) [48p]
  • DB/테이블 생성: CREATE DATABASE, CREATE TABLE (INT/VARCHAR(n)/DATE/FLOAT) [49~52p]

🛠️ TASK 5 DB 연결 → 데이터베이스 & 테이블 생성

⚠️ 먼저 본인 MySQL 서버의 user/password를 준비하세요. 연결 설정은 한 곳(con)에 모아 재사용합니다.
할 일 (server.js에 추가)
  1. mysql require 후 con = mysql.createConnection({...}) 작성 (처음엔 database 없이) [47p]
  2. GET /create-database → CREATE DATABASE mydb 실행 후 "Database created" 응답 [49p]
  3. 연결 설정에 database: "mydb" 추가 [51p]
  4. GET /create-table → customers 테이블 생성 (id INT PRIMARY KEY, name VARCHAR(255), address VARCHAR(255)) [52p]
예상 결과
GET /create-database  → Database created   (콘솔: Connected!)
GET /create-table     → Table created
힌트 (골격)
var mysql = require("mysql");
var con = mysql.createConnection({
  host: "localhost",
  user: /* TODO */,
  password: /* TODO */,
  // database: "mydb"   ← 3단계에서 주석 해제

app.get("/create-table", function (req, res) {
  con.connect(function (err) {
    if (err) throw err;
    var sql = "CREATE TABLE customers (" +
      /* TODO: id, name, address 정의 */
      ")";
    con.query(sql, function (err, result) {
      /* TODO */
    });
  });
});

PRACTICE 6 CRUD (INSERT / SELECT / UPDATE / DELETE) [53~59p]

  • INSERT: INSERT INTO customers (col...) VALUES (val...) [53p]
  • SELECT: SELECT [DISTINCT] 컬럼 FROM 테이블 [WHERE][GROUP BY][HAVING][ORDER BY][LIMIT] — SELECT/FROM만 필수, 순서 고정 [54p]
    • Node에서 결과 result는 행 객체 배열 → res.send(result)로 JSON 전송 [55p]
    • 세 번째 인자 fields = 컬럼 메타데이터(fields[0].name) [56p]
  • WHERE: 조건에 맞는 행만 [57p] ⚠️ 사용자 입력을 문자열에 직접 합치지 말 것(SQL 인젝션) → ? 파라미터 사용
  • DELETE: DELETE FROM customers WHERE 조건 — WHERE 없으면 전체 삭제 [58p]
  • UPDATE: UPDATE customers SET 컬럼=값 WHERE 조건 [59p]

🛠️ TASK 6 customers 테이블 CRUD 완성

할 일 (server.js에 추가 — 각 라우트 = 강의 예제 1개)
  1. GET /insert-customer → (1, 'Company Inc', 'Highway 37') INSERT, "1 record inserted" [53p]
  2. GET /customers → SELECT name, address FROM customers, 결과 JSON 응답 [55p]
  3. GET /customer-field → 같은 SELECT 후 fields[0].name("name") 응답 [56p]
  4. GET /customers-by-address → WHERE address = 'Park Lane 38' 결과 [57p]
  5. GET /update-customer → SET address='Canyon 123' WHERE address='Valley 345', "n record(s) updated" [59p]
  6. GET /delete-customer → WHERE address='Mountain 21', "Records deleted: n" [58p]
예상 결과
GET /insert-customer  → 1 record inserted
GET /customers        → [{"name":"Company Inc","address":"Highway 37"}]
GET /customer-field   → name
GET /update-customer  → 0 record(s) updated   (Valley 345가 없으면 0)
GET /delete-customer  → Records deleted: 0
셀프 체크
  • result.affectedRows는 어떤 값? UPDATE/DELETE에서 왜 유용한가? [58~59p]
  • WHERE 없는 DELETE의 위험성은? [58p]
