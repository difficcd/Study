### ** Radix sort 부가설명 
기수정렬은 계수정렬과 엮어서 이해해둬야 구현이 좋아지므로    
계수정렬을 필수적으로 이해하고 익숙해지도록 해야함    
(삽입정렬, 버블정렬, 합병정렬 사용도 가능하나 기본으로 계수정렬 사용)  

#### 기수 정렬(Radix Sort)   
꼭 계수 정렬(Counting Sort)을 써야 하는 건 아니지만   
안정 정렬(stable sort) 알고리즘을 써야함      
이때 낮은 자리 → 높은 자리 순서로 정렬(LSD 방식)을 하기 때문에    
이전 자릿수의 정렬 결과가 유지되어야 전체가 올바르게 정렬됨  

  
<br> 

 #### 계수 정렬(Counting Sort) 
 구현 코드는 radix&Count_sort.c 참고    
 
* input ; [4, 2, 2, 1]   
1)   count = [0, 1, 2, 0, 1] : 카운트 배열(count)의 크기는 max + 1 = 5 (인덱스 0~4)  
   위의 count 배열의 의미 >>  0은 없음, 1은 1개, 2는 2개, 3은 없음, 4는 1개  

2)   count = [0, 1, 3, 3, 4]  
   위의 count 배열의 의미 >> 결과 배열에서 2는 3번째, 4는 4번째 위치까지 차지함

3)   결과 배열에 배치 (뒤에서부터) : 안정 정렬을 위해 뒤에서부터 순회
     배치 위치(index = count[원소]-1)
<br>

| 순서 | 원소 | count[원소] | 배치 위치 (index = count[원소] - 1) | 결과 (output) |
|------|------|--------------|--------------------------------------|----------------|
| 4번째 (값=1) | 1 | 1 | 0 | [1, _, _, _] |
| 3번째 (값=2) | 3 | 3 | 2 | [1, _, 2, _] |
| 2번째 (값=2) | 2 | 2 | 1 | [1, 2, 2, _] |
| 1번째 (값=4) | 4 | 4 | 3 | [1, 2, 2, 4] |
