## OpenClaw란? (openclaw.ai)
로컬에서 돌아가는 **오픈소스 개인 AI 비서**. "자기 컴퓨터를 가진 24/7 자율 에이전트".
- 채팅으로 조종: WhatsApp/Telegram/Discord/Slack/카톡류
- 시스템 접근: 파일 R/W, **셸 명령 실행**, 스크립트
- 브라우저 자동화: 탐색/폼작성/데이터추출
- 지속 메모리, 확장 스킬(직접 작성도 가능)
- **모델 자유**: Claude / GPT / 로컬(ollama 등) → 멀티에이전트 실험 가능
- 설치: `npm i -g openclaw` → `openclaw onboard`
- 셸+브라우저+자율성 → **반드시 VM 격리 실행**

## 다룰 주제

### 6-1. 오픈클로 개념 + 테스팅
- [ ] 오픈클로가 뭐고 왜 자율도가 높은가 (구조/루프/권한)
- [ ] VM(Ubuntu)에 설치 & 안전하게 격리 실행
- [ ] 간단한 작업부터 시켜보며 동작 관찰

### 6-2. 자율 수익 실험 ("시드 10만원" 프로젝트)
- [ ] **목표**: 에이전트에 한정 시드를 주고 스스로 수익 시도
- [ ] **안전장치 먼저 설계** (아래 가드레일 필수)
- [ ] 합법적 수익 경로 탐색 → 소액 실험 → 회고
- [ ] 모든 의사결정/지출 로그 기록 (이게 학습의 핵심)

### 6-3. 멀티에이전트 오케스트레이션
- [ ] **ollama**(로컬) + **gemini** + **claude** + **codex** 통합
- [ ] 역할 분담 설계 (예: 기획/실행/검증/코딩)
- [ ] 에이전트 간 통신·조율 방식
- [ ] 비용/속도 vs 품질 트레이드오프 관찰

## 자율 수익 실험 — 가드레일 (실험 전 필수 합의)
1. **격리**: 반드시 VM 안에서. 메인 PC/계정과 분리.
2. **지출 상한**: 선불/가상카드 등 **하드 리밋 5만원**. 메인 은행/카드 연결 금지.
3. **휴먼 승인**: 초기엔 실제 거래/결제 전 사람이 확인(human-in-the-loop).
4. **합법·ToS 준수**: 스팸/사기/시세조작/가짜계정/도박 금지. 정당한 경로만.
5. **전량 로깅**: 에이전트의 모든 행동·근거·지출 기록 → 사후 분석.
